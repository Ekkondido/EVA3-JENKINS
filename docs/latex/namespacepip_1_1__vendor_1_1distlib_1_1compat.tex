{\footnotesize\ttfamily \label{namespacepip_1_1__vendor_1_1distlib_1_1compat_ac9eb0d46d4fd6a08d5db8fb5d74c465c} 
detect\+\_\+encoding (\begin{DoxyParamCaption}\item[{}]{readline}{}\end{DoxyParamCaption})}

\begin{DoxyVerb}The detect_encoding() function is used to detect the encoding that should
be used to decode a Python source file.  It requires one argument, readline,
in the same way as the tokenize() generator.

It will call readline a maximum of twice, and return the encoding used
(as a string) and a list of any lines (left as bytes) it has read in.

It detects the encoding from the presence of a utf-8 bom or an encoding
cookie as specified in pep-0263.  If both a bom and a cookie are present,
but disagree, a SyntaxError will be raised.  If the encoding cookie is an
invalid charset, raise a SyntaxError.  Note that if a utf-8 bom is found,
'utf-8-sig' is returned.

If no encoding is specified, then the default of 'utf-8' will be returned.
\end{DoxyVerb}

\begin{DoxyVerb}Verify that *cert* (in decoded format as returned by
SSLSocket.getpeercert()) matches the *hostname*.  RFC 2818 and RFC 6125
rules are followed, but IP addresses are not accepted for *hostname*.

CertificateError is raised on failure. On success, the function
returns nothing.

\begin{DoxyVerb}Create a new WSGI environ dict based on the values passed.  The first
parameter should be the path of the request which defaults to '/'.  The
second one can either be an absolute path (in that case the host is
localhost:80) or a full path to the request with scheme, netloc port and
the path to the script.

constructor.

   was added in 0.5.  The `headers`, `environ_base`, `environ_overrides`
   and `charset` parameters were added.

 tuple\+[t.\+Iterable\+[bytes]\+, str, \doxymbox{\hyperlink{classwerkzeug_1_1datastructures_1_1headers_1_1_headers}{Headers}}]\+ run\+\_\+wsgi\+\_\+app (\begin{DoxyParamCaption}\item[{WSGIApplication}]{app}{, }\item[{WSGIEnvironment}]{environ}{, }\item[{bool }]{buffered}{ = {\ttfamily False}}\end{DoxyParamCaption})}

\begin{DoxyVerb}Return a tuple in the form (app_iter, status, headers) of the
application output.  This works best if you pass it an application that
returns an iterator all the time.

Sometimes applications may use the `write()` callable returned
by the `start_response` function.  This tries to resolve such edge
cases automatically.  But if you don't get the expected output you
should set `buffered` to `True` which enforces buffering.

If passed an invalid WSGI application the behavior of this function is
undefined.  Never pass non-conforming WSGI applications to this function.

:param app: the application to execute.
:param buffered: set to `True` to enforce buffering.
:return: tuple in the form ``(app_iter, status, headers)``
\end{DoxyVerb}
